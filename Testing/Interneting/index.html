So, A Link can be broken down into 3 parts.

  1. The Scheme ie. Https://
  2. The Domain i.e google.com
  3. The Path, elaborating on this, We have a project, let's say Odin project,
    this would become my domain(folder) and everything inside the folder would be the path eg. /en-US/docs/Web/HTML

There are 3 types of links. (For the relative links the scheme and domain are the same)
  • Relative Link
    • I have to make sure to use relative links, I.e can't provide absolute system path,
      the path needs to relative to the file. So in this case i had to use ../../ in order to receive an output
  • Root Relative Link
    • Can only be checked and tested on an actual web server.
  • Absolute Link
    • Links Which are External, and absolute in the sense, nothing we do is going to change that link ever.

Image and image format

    JPG images
  • Used to handle a wide variety of colors with low file size. Does not support transparent pixels.


    • PNG images
  • The file size for the same quality will be bigger than JPG files. But they do support transparent pixels. and no color pallete limitations


    • GIF images
  • Supports animation, but only supports 256 colors, and also supports transparent pixels. Can't have semi-opaque pixels.

    SVG images
  • Scalable Vector Graphics, resolution independent, small file size, supports interactivity and animation.
